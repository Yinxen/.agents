# MiMo TTS 语音合成

MiMo V2 TTS 的核心玩法是**用自然语言括号标记告诉模型"怎么演"**。括号内写情绪/状态/方言，括号外写要念的内容。模型会据此自动调整语速、语调、情感、气口。

## 什么时候使用

- 用户要求把一段文字"说出来"、"念出来"、"读一下"、配音、生成语音/音频文件
- 用户描述了声音风格（如"用撒娇的语气说"、"生气地念"、"温柔点"、"像东北人那样说"）
- 要合成带情感的台词、戏剧化朗读、角色配音

如果用户已经明确告诉你要什么情绪，直接合成；只有在需求模糊（比如只说"帮我念一下这段"，没提语气）且场景看起来需要情感时才追问一句"想要什么感觉的声音？"。别把简单任务复杂化。

## 调用方式

脚本是 `scripts/tts.js`（相对本 SKILL.md 所在目录）。调用时用**本 SKILL.md 所在目录 + `/scripts/tts.js`** 组成绝对路径，或者先 `cd` 到本 SKILL.md 所在目录。推荐前者，避免切换 CWD 带来的副作用。

```bash
node <本 SKILL.md 的目录>/scripts/tts.js \
  --text "（情绪标记）要念的文本" \
  --output /tmp/xxx.mp3
```

例如本 skill 在 `/home/user/.claude/skills/mimo-tts/` 时，脚本路径就是 `/home/user/.claude/skills/mimo-tts/scripts/tts.js`。

默认 voice 是 `default_zh`、格式 `mp3`。不传 `--output` 时自动写到 `/tmp/mimo-tts-<时间戳>.mp3`，避免污染当前目录。

**需要环境变量 `MIMO_API_KEY`**。如果没设置，脚本会打印带申请地址的引导提示（`https://platform.xiaomimimo.com/#/console/api-keys`）——把提示原样转给用户即可，不要自己编一个 key。

### 参数

| 参数       | 简写 | 说明                                      | 默认值                          |
| ---------- | ---- | ----------------------------------------- | ------------------------------- |
| `--text`   | `-t` | **必需**。要合成的文本，可含括号情绪标记  | —                               |
| `--voice`  | —    | 音色：`default_zh` / `default_en` / `mimo_default` | `default_zh`           |
| `--format` | `-f` | 输出格式：`mp3` / `wav`                   | `mp3`                           |
| `--output` | `-o` | 输出文件路径                              | `/tmp/mimo-tts-<时间戳>.<格式>` |
| `--model`  | `-m` | TTS 模型                                  | `mimo-v2-tts`                   |

## 怎么写情绪标记

核心格式：`（状态描述，逗号分隔多个词）正文内容`。状态词可以自由叠加——"生气+东北话+语速快"、"温柔+悄悄话+带叹气"都行。模型的理解力比固定关键词列表强得多，下面的表只是起点，鼓励组合和变体。

### 常见情绪

| 情绪      | 写法                                               |
| --------- | -------------------------------------------------- |
| 撒娇      | `（撒娇，甜美，语速慢）嗯～人家今天好累哦……`       |
| 生气      | `（生气，语速快）你可真行啊你！`                   |
| 温柔      | `（温柔，缓慢）乖，先把这杯温水喝了。`             |
| 悄悄话    | `（悄悄话，非常轻）嘘，轻点儿关门。`               |
| 快乐      | `（开心，兴奋）我们拿到前排票了！`                 |
| 悲伤/抽泣 | `（抽泣）为什么……为什么没有人能看见我的委屈……`     |
| 疲惫      | `（很累，有气无力）让我躺一会儿……`                 |
| 紧张      | `（紧张，呼吸急促）我……我不是故意的……`             |
| 冷漠      | `（冷冰冰）这跟我没关系。`                         |
| 讽刺      | `（阴阳怪气）哦，你可真厉害啊。`                   |

### 非语言声音

| 效果   | 写法                               |
| ------ | ---------------------------------- |
| 咳嗽   | `（咳嗽）咳咳……我好像感冒了。`     |
| 叹气   | `（长叹一口气）唉……算了吧。`       |
| 笑声   | `（突然笑）哈哈哈，你太逗了！`     |
| 打喷嚏 | `（阿嚏）不好意思……`               |
| 停顿   | `我想想……（停顿）嗯……好像是这样。` |

### 方言与角色

| 类型   | 写法                                         |
| ------ | -------------------------------------------- |
| 东北话 | `（东北口音，豪爽）哎呀妈呀，这外头风刮得！` |
| 四川话 | `（四川话）哎哟喂，你还在磨蹭啥子嘛！`       |
| 粤语   | `（粤语）你喺度做乜嘢啊？`                   |
| 台湾腔 | `（台湾腔）你这样很烦耶～`                   |

### 唱歌

用 `<style>唱歌</style>` 包裹（注意是英文尖括号 + 中文"唱歌"）：

```bash
node <skill目录>/scripts/tts.js --text "<style>唱歌</style>我怎么变这样，变得这样倔强？"
```

### 文本本身能传递的信号

不用额外标记，模型也会根据文本形态调节：

| 写法       | 效果                   |
| ---------- | ---------------------- |
| `？`       | 自动升调（疑问）       |
| `！` / `！！` | 加重、情绪增强      |
| `……`       | 犹豫、语气减弱         |
| `我-说-不-行` | 逐字强调            |
| 重复字符（如 `好好好累`） | 语气延长 |

## 组合示例

```bash
# 生气 + 东北话
node <skill目录>/scripts/tts.js \
  --text "（生气，东北话，语速快）你可真行啊你！这事儿你也能干得出来？"

# 温柔 + 悄悄话 + 带叹气
node <skill目录>/scripts/tts.js \
  --text "（温柔，悄悄话，带叹气）唉……你先睡吧，我一会儿就回来。"

# 疲惫
node <skill目录>/scripts/tts.js \
  --text "（疲惫，有气无力）师傅……到地方了叫我一声……"

# 英文
node <skill目录>/scripts/tts.js \
  --text "(excited) We got front-row tickets!" --voice default_en
```

## 心智模型

把 MiMo V2 TTS 当成一个"会演戏的演员"：你用 `（状态描述）+ 正文` 告诉他怎么演，他照着演。状态描述越具体（"生气+语速快+重音在'你'字"），表演越贴近想要的效果。觉得某个情绪表达不到位时，优先尝试换说法或叠加更多状态词，而不是重跑一遍希望它突然开窍。
